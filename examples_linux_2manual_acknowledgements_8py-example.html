examples_linux/manual_acknowledgements.py

This is a simple example of using the RF24 class on a Raspberry Pi to transmit and respond with acknowledgment (ACK) transmissions. Notice that the auto-ack feature is enabled, but this example doesn't use automatic ACK payloads because automatic ACK payloads' data will always be outdated by 1 transmission. Instead, this example uses a call and response paradigm.

Remember to install the Python wrapper, then navigate to the "RF24/examples_linux" folder and open a python3 REPL.
To run this example, enter

1 from manual_acknowledgements import *

and follow the prompts.

Note
this example requires python v3.7 or newer because it measures transmission time with time.monotonic_ns().

122 def slave(count=10):
123  """Polls the radio and prints the received value. This method expires
124  after 6 seconds of no received transmission"""
125  radio.startListening() # put radio into RX mode and power up
126 
127  start_timer = time.monotonic() # start a timer to detect timeout
128  while count and (time.monotonic() - start_timer) < 6:
129  # receive `count` payloads or wait 6 seconds till timing out
130  has_payload, pipe_number = radio.available_pipe()
131  if has_payload:
132  count -= 1
133  received = radio.read(radio.payloadSize) # fetch the payload
134  # use struct.unpack() to get the payload's appended int
135  # NOTE received[7:] discards NULL terminating 0, and
136  # "<b" means its a single little endian unsigned byte
137  payload[0] = struct.unpack("<b", received[7:])[0] + 1
138  # use bytes() to packetize our data into a usable payload
139  # NOTE b"\x00" byte is a c-string's NULL terminating 0
140  buffer = b"World \x00" + bytes([payload[0]])
141  # save response's result
